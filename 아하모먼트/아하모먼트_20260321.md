# 아하모먼트 - Babel → SWC 전환과 번들 크기 감소의 진짜 이유

> 2026-03-21

---

## 발단

> "next.js 환경에서 Babel → SWC로 변경했더니 app.js 파일이 5배 감소했어. 왜일까?"

단순히 SWC가 빠르고 좋아서라고 생각했다.

---

## 탐구 흐름

### 1단계 - 처음 가설: SWC가 코드를 더 잘 줄인다

초기 분석:
- Babel 헬퍼 코드 제거
- styled-components 컴파일 타임 처리
- tree-shaking 효율 향상

→ 맞는 말이지만 **5배 감소를 설명하기엔 부족했다.**

---

### 2단계 - 실제 파일 비교

두 파일을 직접 다운받아 비교해봤다.

| | int (SWC) | prd (Babel) |
|--|--|--|
| 실제 파일 크기 | 8,195KB | 9,832KB |
| Network 전송 크기 | 2,503KB | 10,080KB |

**아이러니:** 실제 파일 크기 차이는 약 1,600KB(약 17%)인데, 네트워크 전송 크기는 4배 차이가 났다.

→ **"왜 다운받은 파일은 비슷한데 네트워크 크기는 다르지?"**

---

### 3단계 - 압축 방식 차이 발견

개발자도구 Network 탭 Response Headers 비교:

- int: `Content-Encoding: br` (Brotli)
- prd: `Content-Encoding: gzip`

→ 처음엔 "Brotli가 gzip보다 압축률이 좋아서 그런 것"이라고 생각했다.

---

### 4단계 - "근데 CloudFront 설정 바꾼 적 없는데?"

Brotli는 CloudFront 설정에서 켜는 건데, 사용자가 설정을 바꾼 적이 없다고 했다.

그런데 **int뿐만 아니라 stg도 Brotli로 바뀌어 있었다.**

→ 공통점 확인:

```bash
git branch --contains b9fa465d  # SWC 커밋
# 결과: develop 브랜치만 포함
```

int와 stg는 develop 브랜치 기반 → SWC 적용됨
prd는 main 브랜치 → SWC 미적용

---

### 5단계 - 진짜 원인 발견 🎯

**CloudFront는 10MB 이상 파일을 압축하지 않는다.**

> AWS 공식 문서:
> *"CloudFront compresses objects that are between 1,000 bytes and 10,000,000 bytes in size."*
> 출처: https://docs.aws.amazon.com/AmazonCloudFront/latest/DeveloperGuide/ServingCompressedFiles.html

```
prd (Babel): 10,068,806 bytes = 10.07MB → 10MB 초과 → 압축 불가
int (SWC):    8,392,262 bytes =  8.39MB → 10MB 이하 → Brotli 압축 ✅
```

**SWC가 파일 크기를 10MB 한도 아래로 내려줬고, 그로 인해 CloudFront Brotli 압축이 자동 활성화된 것이었다.**

---

## 전체 인과관계 정리

```
Babel 빌드
→ 번들 크기 10.07MB (CloudFront 10MB 한도 초과)
→ 압축 안 됨
→ 10,080KB 그대로 전송

       ↓ SWC 전환

SWC 빌드
→ 번들 크기 8.39MB (CloudFront 10MB 한도 이하)
→ Brotli 압축 자동 활성화
→ 2,503KB 전송
```

---

## 수치 정리

| 구분 | 수치 |
|--|--|
| SWC 순수 효과 (파일 크기) | 10.07MB → 8.39MB (약 17% 감소) |
| CloudFront Brotli 압축 효과 | 8.39MB → 2.5MB (약 70% 감소) |
| **최종 전송 크기 감소** | **10,080KB → 2,503KB (75% 감소)** |

---

## 아하모먼트

> SWC가 코드를 잘 최적화해서 번들이 줄었다 → 맞다
>
> 근데 왜 5배나 줄었나? → CloudFront 10MB 압축 한도가 있었다
>
> SWC가 10MB 아래로 내려줬고 → Brotli 압축이 자동으로 켜졌다
>
> 설정을 바꾼 게 없는데 Brotli가 켜진 이유 → SWC가 임계값을 넘어줬기 때문

**단순한 빌드 도구 교체가 인프라 레벨의 최적화를 자동으로 끌어낸 사례.**

---

## 이력서 문장

> Babel → SWC 전환으로 번들 크기를 CloudFront 압축 한도(10MB) 이하로 줄여 Brotli 압축 자동 활성화, 초기 로딩 전송 크기 **75% 감소** (10,080KB → 2,503KB)
